电池压差离群预警_数学模型 

算法名称：P0201 【7*24h监控】【离线预警】电池压差离群预警_数理模型_V2.0
数据来源：tsp和oss
核心逻辑：
step1: 数据初筛
├── min_low_probe_temp ≥ 10℃
├── cell_volt_diff ≥ 1mV
├── 采样间隔 ≥ 3s
├── min_cell_voltage ∈ [2500,4500]mV
└── max_cell_voltage ∈ [2500,4500]mV

step2: 参数投影计算
├── 电流映射：current＿rate = e^{{-A*current}}，A取0.015
├── soc映射：soc＿rate = atan(B*real＿soc^3) *2/ \pi，B取0.002
├── 温度映射：temperature＿rate = 1 / (1 + e^{{-C*avg＿probe＿temperature}})，C取1
├── 压差映射：vdiff＿rate = 1/(D^{{cell＿volt＿diff}} + 1)，D取值逻辑：
│   ├── 场景1：1/(e^(-5/压差)+1)，min_cell_voltage > 0 and max_cell_voltage < 3600mV
│   ├── 场景2：1-0.01×压差，(soc∈[55,65] or soc ≤ 25) and max_volt_diff < 40mV
│   ├── 场景3：0.85，(soc∈[55,65] or soc ≤ 25) and max_volt_diff ≥ 40mV
│   └── 场景4：都不满足取0.4
├── 压差重新计算：Vdiffnew= current＿rate * soc＿rate * temperature＿rate * vdiff＿rate * cell＿volt＿diff

step3: 分位数计算
├── 按battery_type,event_type,soc_tag,curr_tag,cell_type计算single_vdiff_quartiles
├── 按battery_type,event_type,soc_tag,curr_tag,cell_type计算all_vdiff_quartiles
└── 计算同类电池的离群上边缘all_vdiff_upper_bound=round(2.5 * all_vdiff_quartiles_75 - 1.5 * all_vdiff_quartiles_25, 2)

step4: 综合判定
├── cell_type != 'LFP' or soc_tag∈[2,17]
├── 以下任一满足
│   ├── event_type=parking and single_vdiff_quartiles_5 > all_vdiff_upper_bound and single_vdiff_quartiles_95 - single_vdiff_quartiles_5 > all_vdiff_quartiles_95 - all_vdiff_quartiles_5
│   └── soc_tag = 1 and single_vdiff_quartiles_95 > 4 * all_vdiff_quartiles_95 - all_vdiff_quartiles_5 and single_vdiff_quartiles_75 > all_vdiff_upper_bound 

关键参数：
- 空间投影规则

电池温差离群预警_数学模型 

算法名称：P0501/P0706/... 【7*24h监控】【离线预警】电池温差离群预警_数理模型_V1.0
数据来源：block
核心逻辑：
step1: 数据初筛
├── max_temp_diff ≥ 5℃
├── max_discharge_current ≤ 20A
└── abs(max_charge_current) ≤ 20A

step2: 空间投影
    step1 对数据按照环境温度影响函数对温差进行空间投影；
    温度影响函数：
    $$f(T) = (0.5 / (e^{{-A * T1}}+ 0.5))*T2$$
      A为温度系数  ap11_temp_factor  。T1为outside_temp，T2为max_temp_diff。
    step2 分析单个电池的箱线图指标
    对数据按照电池编号battery_id、电池类型battery_type和运行场景event_type分组，计算单电池在不同场景下温差25分位数和最大值。
    step3 计算单电池在不同场景下的电流权重
    对数据按照电池编号battery_id、电池类型battery_type和运行场景event_type分组，计算不同电流区间的持续时长，以及不同电流标签下的权重。电流影响因子函数：
      $$f(I) = e^{{-B * abs(I)}}$$
      B为电流系数  ap11_current_factor  。
    按照电池编号battery_id、电池类型battery_type和运行场景event_type分组，综合电流标签和持续时长计算对应的最终权重；
     $$weight_{new}= sum(weight * duration) / sum(duration)$$
    step4 对单电池在不同场景下的的温差25分位数和最大值按照对应场景的电流影响系数加权；
    step5 计算同类电池在不同运行场景下的箱线图指标，特别是温差箱线图上边缘值；
step3：综合判定
    在相同运行场景下，当单电池的温差加权25分位值 > 同类电池上边缘则判定为温差离群

关键参数：
- 状态：停用
- 算法ID：Algorithm_e0878eaa21

算法名称：P0201/P0202/... 【7*24h监控】【离线预警】诊断层_专家模型_电压类
上游预警层算法：来自以下7个预警层算法的输出结果
- Algorithm_e22dafcf04---75度电池电压不一致预警
- Algorithm_d62367ab45---电池电压采样精度不足_NIO（电池压差离群预警_专家模型2_压差偏高）
- Algorithm_f0a173ec73---block电压不一致_NIO（电池压差离群预警_统计模型）
- Algorithm_103eebf143---电池压差离群_NIO（电池压差离群预警_数学模型）
- Algorithm_4e90fc8f12---alphaQ（电池压差离群预警_统计模型_AlphaQ）
- Algorithm_ap2_vdiff_outlier_empirical_lfp（电池压差离群预警_经验模型_lfp）
- Algorithm_ap3_vdiff_outlier_empirical_ncm（电池压差离群预警_经验模型_ncm）
核心逻辑：
step1: 数据初筛
├── cell_type == 'LFP'
├── 聚合【预警时间+7day】内同battery_id和连续event_type的block数据
└── 对连续相同event_type的block数据进行编号，保留同一编号下，parking_event_duration >= 7200s的事件

step2：数据聚合
├── last_pack_cell_voltage：相同编号值按时间排序后的last_pack_cell_voltage（排除为NULL的block）
├── union_last_pack_cell_voltage：按时间排序后，所有编号值的last_pack_cell_voltage的最后一帧
├── last_highest_cell_voltage：union_last_pack_cell_voltage中数据排序，输出最高电压，不区分电池型号，在所有电芯中计算最高电压。
├── last_lowest_cell_voltage：union_last_pack_cell_voltage中数据排序，输出最低电压，不区分电池型号，在所有电芯中计算最低电压。
└── last_2nd_lowest_cell_voltage：union_last_pack_cell_voltage中数据排序，输出除了最低电压以外的低电压，不区分电池型号，在所有电芯中计算。

step3: 自放电-LFP组诊断
├── 压差阈值判定，任一满足即可
│   ├── 条件1: 
│   │   ├── 3000 <= last_highest_cell_voltage <= 3205 mV
│   │   ├── last_highest_cell_voltage - last_lowest_cell_voltage >= 550 mV
│   │   ├── last_highest_cell_voltage - last_2nd_lowest_cell_voltage <= 550 mV
│   ├── 条件2：
│   │   ├── 3205 <= last_highest_cell_voltage <= 3255 mV
│   │   ├── last_highest_cell_voltage - last_lowest_cell_voltage >= 49 mV
│   │   ├── last_highest_cell_voltage - last_2nd_lowest_cell_voltage <= 49 mV
│   ├── 条件3：
│   │   ├── 3255 <= last_highest_cell_voltage <= 3270 mV
│   │   ├── last_highest_cell_voltage - last_lowest_cell_voltage >= 39mV
│   │   ├── last_highest_cell_voltage - last_2nd_lowest_cell_voltage <= 39 mV
│   ├── 条件4：
│   │   ├── 3270 <= last_highest_cell_voltage <= 3290
│   │   ├── last_highest_cell_voltage - last_lowest_cell_voltage >= 30 mV
│   │   ├── last_highest_cell_voltage - last_2nd_lowest_cell_voltage <= 30 mV
│   ├── 条件5：
│   │   ├── 3290 <= last_highest_cell_voltage <= 3295 mV
│   │   ├── last_highest_cell_voltage - last_lowest_cell_voltage >= 29 mV
│   │   ├── last_highest_cell_voltage - last_2nd_lowest_cell_voltage <= 29 mV
├── 数据准备和过滤
│   ├── 回溯7天的连续parking事件
│   │   ├── 保留：start_time距离第一帧start_time > 7200s的帧 -- 充分去极化
│   ├── 计算last_pack_cell_voltage中的last_highest_cell_voltage和last_lowest_cell_voltage
│   │   └── 保留: 3200 <= last_highest_cell_voltage <= 3289mV
└── lfp_socrate计算
    ├── 电压转SOC: 根据ap28_lfp_soc_ocv_list换算last_highest_cell_voltage和 last_lowest_cell_voltage，得到socmax、socmin
    ├── 计算socgap = socmax - socmin
    ├── 选点逻辑: 找时间不同且电压相近（压差<10mV）的帧
    ├── 线性回归: 得到斜率lfp_socrate (%/day)和r2
    └── 判定: lfp_socrate >= 0.75 OR (lfp_socrate >= 0.3 AND r2 >= 0.16)

step4: 自放电-NCM组诊断  
├── 压差阈值判断
│   │   ├── 3650 <= last_highest_cell_voltage <= 4200 mV
│   │   ├── last_highest_cell_voltage - last_lowest_cell_voltage >= 36 mV
│   │   ├── last_highest_cell_voltage - last_2nd_lowest_cell_voltage <= 15 mV
            OR last_2nd_lowest_cell_voltage - last_lowest_cell_voltage >= 20mV
├── 数据准备和过滤
│   ├── 回溯7天的连续parking事件
│   │   ├── 保留：start_time距离第一帧start_time > 7200s的帧 -- 充分去极化
│   ├── 7天内最后一个有效parking事件的最后一帧avg_volt_diff >= 28mV
│   ├── 保留：last_highest_cell_voltage∈[3650, 4200]mV的帧
└── ncm_socrate计算
│   ├── 电压转SOC: 根据ap28_ncm_soc_ocv_list换算last_highest_cell_voltage和 last_lowest_cell_voltage，得到socmax、socmin
│   ├── 计算socgap = socmax - socmin  
│   ├── 选点逻辑: 找时间不同且电压相近的两帧（电压差<25mV）
│   ├── 线性回归: 得到斜率ncm_socrate (%/day)和r2
│   └── 判定: ncm_socrate >= 0.22

step5: 通用组诊断
├── 计算特征标签
│   ├── event_label: 根据最大充电电流区分快充/慢充/静置/行程
│   ├── cell_type: NCM/LFP
│   ├── soc_tag: SOC区间标签
│   ├── is_event_volt_diff_soc_pcorrelation: 压差SOC正相关性，volt_diff_soc_corr ≥ 0.1
│   ├── is_event_volt_diff_current_pcorrelation: 压差电流正相关性，volt_diff_current_corr ≥ 0.5
│   ├── is_event_volt_diff_entropy: 压差熵增, entropy(volt_diff_quartiles) ≥ 0.5
│   ├── is_vdiff_high_alarm: 压差大,avg_volt_diff ≥ 40 mV
│   ├── is_high_outlier_alarm: 单体电压高离群，max_volt_diff_max_cell_voltage ≥ mean + 4.5*std
│   ├── is_low_outlier_alarm: 单体电压低离群，max_volt_diff_min_cell_voltage ≤ mean - 4.5*std
│   ├── is_extre_unstable: 极值不稳定性
│   │   ├── size(max_volt_sn_count) >= 50
│   │   ├── size(min_volt_sn_count) >= 50
│   │   ├── size(max_volt_sn_count) / block_msg_count <= 0.8
│   │   ├── size(min_volt_sn_count) / block_msg_count <= 0.8
│   ├── hg_sn_is_busbar: Busbar异常，检查block内的最高电压电芯编号是否在busbar
│   ├── is_module_imbalance: 支路电压失衡，block内最高最低电芯不在同一支路，但属于同一环路
│   ├── is_loop_volt_low_outlier: 环路电压低离群
│   │   ├── block内模组最低电压 < 总平均电压 - 2.5 * std
│   │   ├── block内模组最低电压 < 总平均电压 - 240
│   ├── is_circ: 环流，问题电芯所在支路为支路1，另一支路为支路2，满足支路2最大电压 < 支路1次低电压
│   └── is_selfdch: 自放电
│   │   ├── block内存在环流现象
│   │   ├── block内按照soc/事件类型分组后计算压差扩大速率(单位:mV/day)
│   │   ├── block最大压差≥36mV
└── 诊断逻辑匹配
    ├── P0201 自放电大: NCM + 单体电压低离群=1 + 自放电=1
    ├── P0202 欧姆内阻大: NCM + 快充 + 压差电流正相关性=1 + 压差熵增=1 + 单体电压高离群=1
    ├── P0203 极化内阻大: NCM + 充电 + 压差SOC正相关性=1 + 压差熵增=1 + 单体电压高离群=1
    ├── P0204 串联busbar异常: NCM + 快充 + 压差电流正相关性=1 + 压差熵增=1 + 单体电压高离群=1 + busbar异常=1
    ├── P0205 并联busbar异常: NCM + 快充 + 单体电压高离群=1 + 单体电压低离群=1 + busbar异常=1 + 支路电压失衡=1
    ├── P0206 电池电芯低容: 快充 + SOC≥18 + 压差SOC正相关性=1 + 压差熵增=1 + 单体电压高离群=1
    ├── P0207 CSC功耗异常: NCM + parking/slow_charge + 自放电=1
    ├── P0208 EMC干扰: 快充 + 极值不稳定性=1
    ├── P0301 CSC电压采样误差大: parking + 2≤soc_tag≤16 + 单体电压高离群或低离群=1
    └── P0309 SOC偏差大: charge + 4≤soc_tag≤16 + 压差大=1

step6: LFP组+NCM组+通用组融合
├── 融合规则（从上往下匹配，命中即退出）
│   ├── 规则1: LFP组报P0201 → 输出P0201
│   ├── 规则2: NCM组报P0201 → 输出P0201  
│   ├── 规则3: 通用组报故障≠P0201 → 输出通用组结果
│   └── 规则4: 通用组报故障=P0201 → 输出P0201
└── 融合结果补充判定
    ├── avg_volt_diff: max_volt_diff预警时间所在block的数据
    ├── vmax_to_mean_rest_diff: 最大电压值与刨除最大电压后平均值的差
    └── 特殊处理: algorithm_id = alphaQ时，diagnosis_code直接赋值为P0201

step7: 诊断输出
└── 输出融合结果中的诊断代码和故障模式

核心逻辑：
step1: 数据初筛与输入准备
├── FROM: 预警层数据表 dm_battery_alarm.d_i_alarm_results
├── DO: block特征表 dm_battery_profile.d_i_battery_block_features  
├── TO: 诊断层数据表 dm_battery_compass.d_i_battery_diagnosis_results
├── 运行周期: 按天运行
└── 算法范围: 来自3个预警层算法
    ├── Algorithm_ec97501447 --- block温度不一致_NIO
    ├── Algorithm_e0878eaa21 --- 电池温差离群_NIO
    └── Algorithm_c1bb911a64 --- block电池温差异常_NIO

step2: 追溯与筛选
├── 加载block数据，数据时间为【预警时间+ap27_date_back】内
├── 直接引用的数据：
│   ├── battery_id
│   ├── start_time
│   ├── end_time
│   ├── max_temp_diff
│   ├── min_low_probe_temp
│   ├── event_type
│   ├── avg_temp_diff
│   ├── max_temp_diff_min_probe_temp_sn
│   ├── max_temp_diff_max_probe_temp_sn
│   └── last_pack_probe_temp
└── 对每个battery_id，在时间范围【预警时间-ap27_date_back】~【预警时间】范围内，按start_time进行升序排序

step3: 参数解析
└── 见5.参数配置

step4: 诊断步骤
├── Step1 分组
│   ├── 每个battery_id里回溯时间窗口期内每一帧的last_pack_probe_temp温度列表中，最高温度记为Max_temp，最低温度记为Min_temp
│   ├── 计算每一帧的最大温差Delta_temp = Max_temp - Min_temp
│   ├── 对每一帧定义异常标记：若Delta_temp > ap27_max_temp_diff_limit，则本帧记为异常帧（flag = True），否则记为正常帧（flag = False）
│   ├── 对比当前帧与上一帧的异常标记：若当前帧flag与上一帧不同，或当前帧无上一帧，则该帧为一个新组的起始帧
│   └── 将分组结果划分为两类：
│       ├── 疑似组：组内帧满足 Delta_temp >= ap27_max_temp_diff_limit
│       └── 正常组：组内帧满足 Delta_temp < ap27_max_temp_diff_limit
│
├── Step2 压差持续时间判断
│   ├── 对于疑似组，对本组持续时长进行判断。每组持续时长last_duration=本组最后一帧的end_time - 本组第一帧start_time
│   ├── 若对于某个battery_id的回溯时间内，存在至少一个疑似组同时满足以下两个条件则该battery_id的预警时间当天的预诊断pre_diagonsis_code=P0302；否则pre_diagonsis_code=diagnosis_code=P0308：
│   │   ├── 每组持续时长last_duration >= ap27_temp_diff_lasts_limit
│   │   └── 在该疑似组内，满足(Min_temp > ap27_min_low_temp_threshold 的帧数) 占该组（总帧数）的比例 >= ap27_min_low_temp_ratio
│   └── pre_diagonsis_code=P0302的电池进入Step3判断
│
├── Step3 恢复判断
│   ├── 进入step3的电池的疑似组中，选择结束时间最接近预警时间的一个疑似组，且该疑似组内帧数 ≥ 3
│   ├── 计算指标：最低温度变化幅度delta_low_temp，最低温度变化斜率temp_slope_i，和负斜率占比negative_temp_slope_ratio
│   │   ├── 最低温度变化幅度delta_low_temp = 本疑似组最后一帧的Min_temp_last - 本疑似组第一帧的Min_temp_first
│   │   ├── 最低温度变化斜率temp_slope_i = Min_temp[i] - Min_temp[i-1]，计算方法为逐帧差分，所以注意temp_slope_i的总次数=组内帧数-1
│   │   └── 负斜率占比negative_temp_slope_ratio = (temp_slope_i < 0的次数) / (temp_slope_i 总次数)
│   └── 条件判断：若同时满足以下三个条件，则认为模型温度恢复 - 输出diagnosis_code=P0308，否则保留 diagnosis_code=pre_diagnosis_code=P0302：
│       ├── 条件1：最低温度变化幅度delta_low_temp >= ap27_min_low_temp_range
│       ├── 条件2：本疑似组最后一帧的最低温度Min_temp_last > ap27_min_low_probe_temp_last_threshold
│       └── 条件3：负斜率占比negative_temp_slope_ratio < ap27_negative_temp_slope_ratio_threshold

step5: 诊断输出
├── 输出diagnosis_code结果到diagnosis_code
├── 带出到diagnosis_features（表dm_battery_compass.d_i_battery_diagnosis_results中字段）的字段：
│   ├── 输出每个battery_id最后一个疑似组的最后一帧Delta_temp，到diagnosis_features（输出名字为last_suspect_Delta_temp）
│   ├── 输出每个battery_id最后一个疑似组的last_duration，到diagnosis_features（输出名字为last_suspect_last_duration）
│   ├── 输出每个battery_id最后一个疑似组最低温度变化幅度delta_low_temp（输出名字为last_suspect_Delta_Min_temp）
│   ├── 输出每个battery_id最后一个疑似组最后一帧的最低温度Min_temp_last（输出名字为last_suspect_Min_temp）
│   └── 输出每个battery_id最后一个疑似组负斜率占比negative_temp_slope_ratio（输出名字为last_suspect_negative_temp_slope_ratio）
└── 需要输出到shield页面【特征】位置的字段：
    ├── 每个battery_id最后告警触发时刻的event_type为alarm_msg_type，中文名字为：事件类型：alarm_msg_type（parking翻译为静置，charge翻译为充电，journey翻译为行驶）
    ├── 把最后一个疑似组的avg_temp_diff降序，start_time升序，然后取第一行帧的Delta_temp，中文名字：最大温差：Delta_temp
    └── 每个battery_id的最后一个疑似组里的问题电芯编号fault_probe_sn，中文名字为问题温度采样点：fault_probe_sn，判定问题电芯编号逻辑如下：
        └── 若negative_temp_slope_ratio > 0.5，则fault_probe_sn = max_temp_diff_max_probe_temp_sn；若negative_temp_slope_ratio <= 0.5，则fault_probe_sn = max_temp_diff_min_probe_temp
关键参数：
- 时间参数：
